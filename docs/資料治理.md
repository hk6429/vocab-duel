# 字鬥英雄（vocab-duel）資料治理文件

版本：2026-07-18　適用對象：學校資訊組 / 導師 / 家長委員會 需要「據以簽核使用」層級的說明時參考。
本文件比 `privacy.html`（給一般使用者的白話說明）更正式、更技術，逐條對照**實際程式碼**撰寫，如與程式碼不一致以程式碼行為為準，請回報作者更正。

---

## 0. 系統背景

- 服務名稱：字鬥英雄（vocab-duel），一位國中國文老師（陳乃誠）利用課餘時間開發維護的免費個人教學專案，**非商業產品、非學校官方系統**。
- 前端：純靜態 HTML/JS，無帳號、無登入、無密碼。
- 後端：Cloudflare Pages Functions（`functions/api/*.js`），資料庫為 **Cloudflare D1**（SQLite），透過 `functions/api/_redis.js` 這層 shim 模擬 Redis 常用指令（`get/set/hset/lpush/expire...`），實際落地儲存皆在 D1 的 `kv` / `hash` / `list` / `zset` 資料表。
- 部署站台：`vocab-duel.vercel.app` / `vocab-duel.pages.dev` / `vocab-duel.netlify.app`（三個鏡像站，資料庫共用同一個 Cloudflare D1）。

## 1. 資料清單（欄位 × 儲存位置 × 保存期限）

以下逐一對照 `functions/api/*.js` 原始碼中的實際 key 前綴、TTL 設定值（`ex:` 秒數 / `redis.expire(...)`）撰寫，**未捏造**。TTL 到期由 `_redis.js` 惰性過期機制清除（讀取時比對 `exp`，過期即視為不存在；未讀取的過期列不會立刻物理刪除，但對外一律不可再存取）。

| 資料內容 | Key 前綴（來源檔） | 觸發時機 | 保存期限（TTL） |
|---|---|---|---|
| 個人雲端存檔（進度 blob：學習進度、等級、字幣、寵物、城鎮存檔等整包 JSON，內容不透明、由前端決定格式） | `vd:sync:{同步碼}`（`sync.js`） | 使用者主動輸入 6–12 碼同步碼「上傳」時才寫入 | **400 天**（`TTL = 400*24*60*60`），逾期未使用自動清除；單筆上限 512KB |
| 單字之城（城鎮）存檔 | `vd:town:{code}`（`town.js`） | 使用者主動存城鎮 | **400 天**（2026-07 起已補上 TTL，與 sync.js 一致；**注意**：`privacy.html` 目前仍寫「沒有到期設定」，屬舊版說明，本文件以程式碼為準，已列入待修） |
| 單字之城參觀碼對照（本體碼↔6碼參觀碼） | `vd:town:visit:{v}` / `vd:town:visitof:{code}`（`town.js`） | 同上，存城時冪等產生 | 400 天，跟隨城鎮存檔 |
| 單字之城訪客留言簿（暱稱＋表情，最多留 20 筆） | `vd:town:guest:{code}`（`town.js`） | 訪客用參觀碼留言 | 隨 `vd:town:*` 系列一併管理（LPUSH 清單，無獨立 TTL 標記，實務上依城鎮存檔生命週期） |
| 城鎮贈禮防重複標記 | `vd:town:gift:...`（`town.js`） | 每日贈禮 | 90000 秒（約 25 小時，防同日重複領取） |
| 班級排行榜（暱稱、等級、字幣、掌握字數、本週新掌握字數等戰績） | `vd:board:{班級碼}`（`board.js`） | 學生主動同步戰績到班級碼 | **240 天**（約一學年） |
| 班級認領資料（PIN、班級名稱、認領時間） | `vd:class:{班級碼}`（`class.js`） | 老師認領班級碼 | **240 天** |
| 班級字表指派（最多 8 筆指派內容） | `vd:class:{班級碼}:asg`（`class.js`） | 老師發布指派 | **240 天** |
| 全班弱字聚合（單字→錯誤次數統計，**不含個人身分**，是全班加總數字） | `vd:weak:{班級碼}`（`class.js`） | 學生端批次上報錯字次數 | 未見獨立 TTL 設定，隨 `vd:class:*` 家族管理（沿用 240 天量級） |
| 即時對戰房間（雙方對戰狀態快照） | `vd:room:{code}`（`room.js`） | 開房/加入即時對戰 | **600 秒**（10 分鐘），下課即蒸發 |
| 非同步挑戰書（發戰帖暱稱、成績、seed、題目範圍） | `vd:room:ch:{code}`（`room.js`） | 發戰帖 | **7 天**（`CH_TTL = 7*86400`） |
| 隨堂即時模式（老師開場的班級碼+PIN、題目字表、學生名冊、逐題作答記錄、暱稱） | `vd:live:{code}` / `vd:live:{code}:p`（`live.js`） | 老師開場「隨堂即時」 | **3600 秒**（1 小時），下課自動蒸發 |
| 裝備市場掛單（賣家暱稱、裝備數值、價格、HMAC 簽章、claimKey） | `vd:market:item:{id}`（`market.js`） | 學生上架裝備 | **7 天**（`ITEM_TTL = 7*86400`） |
| 每日限購次數標記（暱稱＋日期） | `vd:market:buys:{nick}:{日期}`（`market.js`） | 買家購買 | 首次購買後設 **86400 秒**（1 天） |
| 詞靈影子 PvP 快照池（暱稱、寵物 id、戰力等，用於配對對手） | `vd:petpool`（`pets.js`） | 使用者上傳出戰寵快照 | 依 LTRIM 保留最新 200 筆（無時間 TTL，靠筆數上限輪替） |
| 詞靈全站排行榜（暱稱→積分 ZSET） | `vd:petboard:global`（`pets.js`） | 同上 | 無 TTL，Top 50，同暱稱只留一筆（覆寫） |
| 詞靈排行榜附加資訊（暱稱→寵物名/等級） | `vd:petboard:meta`（`pets.js`） | 同上 | 無 TTL |
| 傳承銘文 UGC（學生自編例句，匿名分享） | `vd:lore:*`（`lore.js`） | 學生投稿例句 | 審核通過後保留 **400 天**（`60*60*24*400`），未通過審核者停留於審核佇列 |
| 班級語錄牆（暱稱＋自編例句，最多 200 筆） | `vd:quote:{code}`（`quotes.js`） | 學生發布語錄 | **240 天** |
| 單字發音/解釋錯誤回報 | 無（`report.js`） | 使用者送出回報 | **不落地存 D1**，純轉發到老師的 Telegram，D1 端無任何保存 |
| 各類限流計數器（IP → 次數，防洗版/濫用） | `vd:rl:lore:{ip}` / `vd:rl:report:{ip}` 等各檔內同款 | 每次寫入型請求 | 60～300 秒（各檔各異，見程式碼內 `rateLimited` 函式） |

**共通說明**：
- 所有寫入均需使用者「主動」操作觸發（輸入同步碼／班級碼／暱稱並按下上傳、分享、發表等按鈕），系統**不會**背景自動蒐集或上傳任何本機資料。
- 暱稱／班級碼／同步碼皆為使用者自訂字串，系統不驗證、不對應到真實姓名或學號，也**沒有帳號系統**，因此系統本身無法回答「這個同步碼是哪個學生」。
- 黑名單過濾：暱稱、城名、訪客留言、班級語錄皆會擋掉常見中英文辱罵字詞（見各檔 `BAD_WORDS` 正則），但非窮舉、不做完整內容審核。
- 目前**沒有**任何第三方追蹤/廣告 SDK（無 GA、無 FB Pixel），僅有 visitor-badge（訪客計數圖示）與 GoatCounter（純網站流量統計，不綁定個人）。

## 2. 刪除方式

### 2.1 使用者端自助刪除
- **本機資料**：清除瀏覽器該網站的網站資料（localStorage）即可清空本機端所有進度，或使用無痕視窗遊玩不留本機紀錄。
- **雲端資料（一般使用者）**：因無帳號系統，使用者可自行覆寫同一個同步碼/班級碼的資料（等同於「蓋掉舊資料」），或單純不再使用、放著讓 TTL 自然到期清除。系統**無法**主動辨識「這是哪一位真人」故無法提供傳統的「刪除我的帳號」入口。
- 使用者如需提前處理／有疑慮，可透過學校聯繫本站維護者（大乃老師），提供其使用的同步碼／班級碼／暱稱，由維護者手動以下方 wrangler 指令個別清除。

### 2.2 班級解散／學年結束後，資訊組協助清 D1

若學校要求「班級解散後立即清除該班所有雲端資料」，可由具備 Cloudflare 帳號權限的資訊組人員，透過 `wrangler d1 execute` 直接對指定 key 前綴做清除。範例（`{班級碼}` 換成實際班級碼）：

```bash
# 先查有哪些 key 屬於這個班級碼（確認範圍再刪，避免誤刪其他班）
npx wrangler d1 execute vocab-duel-db --remote \
  --command "SELECT k FROM kv WHERE k LIKE 'vd:class:{班級碼}%' OR k LIKE 'vd:board:{班級碼}' OR k LIKE 'vd:weak:{班級碼}' OR k LIKE 'vd:quote:{班級碼}' OR k LIKE 'vd:live:{班級碼}%';"

# 確認無誤後執行刪除（kv 表存 string 型資料）
npx wrangler d1 execute vocab-duel-db --remote \
  --command "DELETE FROM kv WHERE k LIKE 'vd:class:{班級碼}%' OR k LIKE 'vd:board:{班級碼}' OR k LIKE 'vd:weak:{班級碼}' OR k LIKE 'vd:quote:{班級碼}' OR k LIKE 'vd:live:{班級碼}%';"

# 語錄牆／訪客留言簿等資料存在 list 表（LPUSH 系列），一併清除
npx wrangler d1 execute vocab-duel-db --remote \
  --command "DELETE FROM list WHERE k LIKE 'vd:quote:{班級碼}' OR k LIKE 'vd:town:guest:%';"
```

- `database_name = "vocab-duel-db"`、`database_id = "547a34b7-30ee-42f4-99f2-49b783c3339a"`（見 `wrangler.toml`），資訊組如需自行操作，須先取得 Cloudflare 帳號存取權限（目前由作者本人持有）。
- 個人同步碼（`vd:sync:{code}`）、單字之城（`vd:town:{code}`）等**非班級性質**資料，因不知道對應到哪位學生，建議請該生自行告知同步碼後，比照上方語法對單一 key 做 `DELETE`。
- 若學校無 Cloudflare 存取權限，可請作者本人（大乃老師）代為執行，並回覆執行結果作為紀錄。

## 4. 未成年使用建議與最小化原則

1. **最小化蒐集**：系統設計上刻意不要求註冊、不要求真實姓名、不做任何身分驗證，僅蒐集遊戲運作所必需的暱稱、進度、班級碼／同步碼。
2. **暱稱非真名**：務必提醒學生使用綽號、座號或代號，不要填寫真實姓名——系統無法驗證，需仰賴使用者與導師的教育與自律。
3. **社交曝光面知情**：以下功能會讓學生輸入的暱稱被同班同學（甚至知道班級碼／參觀碼的任何人）看到，建議家長/導師第一次使用時陪同了解：班級排行榜、即時對戰／隨堂即時模式的名冊、單字之城訪客留言簿、班級語錄牆、傳承銘文（UGC 例句）。
4. **建議由導師統一管理班級碼與 PIN**：班級碼／PIN 屬於「知道就能寫入」等級的輕量驗證，非高強度帳號安全機制，請導師勿在班級碼/PIN 洩漏疑慮下沿用同一組給下一屆使用。
5. **資料最小保留**：即時對戰房間（10 分鐘）、隨堂即時模式（1 小時）等高時效性資料，系統已設計成下課即自動蒸發，不做長期保留。
